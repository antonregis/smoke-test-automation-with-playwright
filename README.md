# Smoke Test Automation with Playwright

## About the Project
This project demonstrates 5 smoke test cases for an e-commerce website using the Page Object Model Approach. Below is the list of test cases for the smoke test:

| # | Test Case | Expected Outcome |
|:--|:----------|:-----------------|
|1  | Homepage loads and displays correctly | Site is accessible, key elements such the header, navigation, and banners render |
|2  | User can search for a product and see results | Search functionality works, product listings display |
|3  | User can view product details page | Product page loads with image, price, description, "Add to Cart" button | 
|4  | User can add a product to cart and view cart | Add to Cart works, cart displays correct item/quantity/price |
|5  | User can complete checkout (up to payment page) |Cart → Checkout form → Payment page flows without errors |

## Configuration Notes
The playwright.config.js file has been customized for demonstration purposes:
- Slow motion enabled (slowMo: 1000) to make browser actions easier to observe
- Browser positioned on Screen 3 (center) for multi-monitor setups
- Viewport fixed at 1280x720
- Only Chromium is enabled by default

## Test website
The tests target a local Drupal Commerce Kickstart site available at https://playwright.ddev.site/. The environment is containerized using Docker and managed with DDEV, with all components running locally.
